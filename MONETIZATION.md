# MONETIZATION.md

## CardCue Monetization Strategy

### Core principle
CardCue should stay a **small, simple credit-card utility**.  
Monetization must **not** make the product feel cluttered, annoying, or manipulative.

The monetization model should preserve the same product philosophy as the rest of the app:

- minimal UI
- low cognitive load
- no upsell noise
- no feature bloat
- no “growth hack” behavior

---

# 1. Current policy

## Launch phase
CardCue is currently **free to use**.

During the early phase, the goal is:

- get real users
- collect feedback
- build trust and ratings
- validate that people actually use CardCue over time

CardCue should **not** introduce monetization pressure too early.

---

# 2. Long-term monetization model

## Final monetization model
CardCue should monetize with a **single one-time purchase** tied to wallet size.

### Free tier
- User can add **up to 10 cards**
- Free tier includes the **full CardCue product experience**
- No ads
- No subscription
- No feature gating

### Paid unlock
- If the user wants to add an **11th card**, show a paywall
- Offer a **one-time $0.99 purchase**
- Purchase unlocks **unlimited cards permanently**

---

# 3. Hard product rules

## Rule 1 — Free tier must remain a complete product
This is the most important rule.

The free version must still feel like a **complete, honest, usable app**, not a crippled demo.

Free users should still get the full core experience, including:

- My Cards
- Best Card / recommendation flow
- Quick Reference
- Cue
- Notes
- rotating category updates
- normal card editing
- normal card usage flow

The only free-tier limitation should be:

> **Maximum 10 cards**

That is the only monetization boundary.

---

## Rule 2 — No feature fragmentation
Do **not** split core functionality into free vs paid versions.

### Not allowed
Examples of things CardCue should **not** do:

- lock Notes behind payment
- lock Cue behind payment
- lock Quick Reference behind payment
- make recommendations worse for free users
- hide rotating category updates from free users
- show incomplete card information unless user pays
- limit basic editing unless user pays

This would damage trust and break CardCue’s design philosophy.

---

## Rule 3 — No ads
CardCue should **never** use ads.

Reasons:

- ads directly conflict with the app’s minimalist design
- ads add visual noise and cognitive overhead
- ads make the product feel cheap and cluttered
- even a simple app loses its simplicity if it constantly shows ads

CardCue should remain:

- clean
- quiet
- utility-like
- focused

---

## Rule 4 — No subscription
CardCue should **not** use a recurring subscription model.

Reasons:

- CardCue is a lightweight utility, not a subscription service
- the product value comes from simplicity and convenience, not ongoing premium content
- subscriptions would create pressure to invent unnecessary “Pro” features
- subscriptions are too heavy for the kind of value CardCue provides

The correct payment model is:

> **small one-time unlock**

---

## Rule 5 — No aggressive upgrade prompts
CardCue should not behave like a growth app.

Do **not** add:

- upgrade banners on the home screen
- recurring “go premium” prompts
- upgrade nags in settings
- upgrade popups on app launch
- upgrade reminders after normal usage

The user should not be thinking about monetization during normal use.

---

# 4. Paywall trigger

## The paywall appears only at one moment
The paywall should appear **only when the user attempts to add the 11th card**.

That means:

- user currently has 10 cards
- user taps “Add Card”
- user tries to proceed with adding another card

Only then should CardCue show the unlock screen.

### Important
There should be **no proactive upsell anywhere else in the app**.

This is a hard UX rule.

---

# 5. Why the 10-card limit works

The 10-card limit is a good monetization boundary because it matches actual user behavior.

## Light users
Many users only have:

- 2 cards
- 3 cards
- 5 cards
- maybe 6 cards

These users can use CardCue for free forever.

## Heavy users
The people most likely to have:

- 10+ cards
- category overlap
- multiple rotating cards
- airline / hotel / grocery / dining optimization setups

are also the users who get the most value from CardCue.

So the monetization logic becomes:

> CardCue is free for most people.  
> If you manage a larger wallet, pay a tiny one-time fee to unlock unlimited cards.

This feels fair, simple, and easy to explain.

---

# 6. Paywall copy principles

The unlock screen should feel calm and honest.

It should **not** feel like a sales funnel.

## Tone
The copy should be:

- short
- plain
- respectful
- non-pushy

Avoid:

- “limited time”
- fake urgency
- premium lifestyle marketing
- dark patterns
- manipulative upgrade language

---

# 7. Recommended paywall copy

## Title
**Unlock unlimited cards**

## Body
CardCue is free for up to **10 cards**.  
If you manage a larger wallet, you can unlock **unlimited cards** with a **one-time $0.99 purchase**.

No ads. No subscription.

## Actions
Primary button:
- **Unlock unlimited cards — $0.99**

Secondary buttons:
- **Not now**
- **Go back**

---

# 8. UX behavior around the paywall

## Desired behavior
If the user hits the 10-card limit and tries to add another card:

1. Show the unlock screen
2. Explain the rule clearly
3. Offer the one-time purchase
4. Allow the user to dismiss the paywall
5. Return them cleanly to the previous screen if they choose not to purchase

## Avoid
Do not make the user feel trapped.

Examples to avoid:

- repeated forced prompts
- hiding the cancel button
- multiple upsell steps
- fake “discount” language
- making the app feel broken unless the user pays

---

# 9. What CardCue should never become

CardCue should not become a “monetization-first” app.

It should not evolve into:

- a subscription dashboard
- a premium upsell machine
- an ad-supported finance app
- a feature-bloated rewards tracker
- a product that constantly asks users for money

CardCue’s advantage is **clarity and restraint**.

Monetization should support the app, not reshape it.

---

# 10. One-sentence product rule

If a future monetization idea makes CardCue feel more annoying, noisy, or complex, it is probably the wrong idea.

---

# 11. Final monetization summary

## CardCue monetization policy
- CardCue is **free** for up to **10 cards**
- The free tier remains a **complete product**
- The **only** free-tier limitation is card count
- Adding an **11th card** triggers the unlock paywall
- Unlimited cards are unlocked with a **one-time $0.99 purchase**
- **No ads**
- **No subscriptions**
- **No feature gating**
- **No aggressive upsell UI**
